Library Management System

Flask · SQLite · Ubuntu VM

School Seminar Presentation

Python Flask SQLite VirtualBox

The Problem

  • School libraries still use paper-based or spreadsheet tracking for books and loans
  • Manual processes are slow, error-prone, and don't scale
  • Commercial library systems are over-engineered for small institutions
Goal: Build a minimal, defensible MVP — a working web application that can manage a book catalog, issue and return loans, and be deployed inside a single Ubuntu VM for demonstration.

Technology Stack

Modular Service Structure

Not microservices. A modular monolith — separate Python packages, one Flask process. 

library-system-school/
├── apps/
│   └── web_app/              ← Flask UI, routes, templates
├── services/
│   ├── auth_service/         ← admin login, session management
│   ├── catalog_service/      ← book CRUD, inventory tracking
│   └── circulation_service/  ← issue / return transaction logic
├── shared/                   ← DB config, contracts, utilities
└── scripts/                  ← seed_demo, bootstrap_ubuntu

Each service is a narrow Python package. All run in-process, keeping deployment simple.

Deployment on Ubuntu VM

# Clone
git clone https://github.com/ezekaj/library-system-school.git
cd library-system-school

# Environment
python3 -m venv .venv
source .venv/bin/activate

# Install
pip install -r requirements.txt

# Seed demo data
python -m scripts.seed_demo

# Run
python -m apps.web_app

Or one-step: bash scripts/bootstrap_ubuntu.sh — installs Python, creates venv, installs deps, seeds data.

Default admin: admin@library.local / admin123

Problems & Fixes

Problem Root Cause & Fix
ModuleNotFoundError on fresh Ubuntu VM System Python has no Flask or SQLAlchemy. Fixed by installing dependencies via requirements.txt inside a venv.
python3-venv package missing on Ubuntu Ubuntu ships Python without venv by default. Fixed with sudo apt install python3-venv in the bootstrap script.
CSS and static assets not loading Flask static_url_path was misconfigured. Fixed by using Flask's default /static endpoint and correct folder layout.
Seed script not importable from CLI Needed if __name__ == "__main__" guard and proper module-level imports. Made runnable as python -m scripts.seed_demo.

Conclusion

  • The system runs end-to-end: login → catalog → issue → return
  • The modular structure is easy to explain and extend in class
  • Deployment fits inside a single Ubuntu VM — reproducible for any seminar
