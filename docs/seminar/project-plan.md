# Project Plan

## Phase 1: Foundation

- Initialize repository
- Define architecture and scope
- Create folder structure for app, services, and shared code

## Phase 2: Data Model

- Define tables for users, books, copies, and loans
- Set up SQLite connection and database initialization
- Add seed data for the seminar demo

## Phase 3: Core Features

- Admin login
- List books
- Add books
- Issue books
- Return books

## Phase 4: Presentation Readiness

- Generate ERD
- Run the application inside Ubuntu VM
- Capture screenshots of setup and application workflow
- Prepare a concise seminar deck

## MVP Acceptance Criteria

- Admin can sign in successfully
- At least three books exist in the system
- A book can be issued and returned
- Database state changes are visible
- The system runs on Ubuntu in a virtual machine
- The architecture and ERD can be explained in class
